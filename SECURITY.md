# Security Policy

## Supported Versions

This project is currently under active development and has not had an official stable release yet.

At this time, only the latest code in the `main` branch is considered supported for security-related fixes.

| Version | Supported |
| ------- | --------- |
| `main`  | ✅        |
| pre-release / older commits | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately and do **not** open a public issue.

### How to report
Please contact the maintainer privately via one of the following methods:

- Email (if provided in the repository profile or documentation)
- GitHub direct contact / private communication channel

If no private contact method is available, please open a minimal public issue **without disclosing sensitive details**, asking for a private contact channel.

### What to expect
- I will try to acknowledge receipt of your report within **7 days**
- I will review and assess the report as soon as possible
- If the issue is confirmed, I will work on a fix and publish it in the `main` branch
- If needed, I may ask for additional details or reproduction steps
- Once resolved, I may document the fix in the commit history or release notes

### Scope
Please report vulnerabilities related to:

- exposed API keys, tokens, or secrets
- unsafe handling of configuration files
- insecure storage of credentials
- command execution risks
- unsafe file access or path handling
- dependency-related security issues
- any behavior that could expose user or system data

Thank you for helping improve the security of this project.
