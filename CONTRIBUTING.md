# Contributing

First of all, thank you for your interest in this project.

## Project Status

This project is primarily a **personal challenge** and a learning-focused project.

My goal is to build and improve it **independently**, so at this stage I am **not actively looking for direct code contributions or large collaborative development**.

That said, I truly appreciate interest from others, and I am happy to receive:

- ideas for improvements
- bug reports
- feature suggestions
- documentation fixes
- small corrections
- feedback on code quality, structure, or usability

## What I Welcome

You are very welcome to open an issue for:

- bugs or unexpected behavior
- suggestions for new features
- usability improvements
- code cleanup ideas
- documentation corrections
- compatibility notes
- edge cases I may have missed

Constructive feedback is always appreciated.

## Pull Requests

Because this project is intended as a personal challenge, I may be selective about pull requests.

In general:

- **Small fixes** may be accepted
- **Documentation improvements** are welcome
- **Bug fixes** may be accepted if they are clear and focused
- **Large refactors or major feature implementations** may be declined, even if they are good, because I want to build most of the project myself

Please do not take it personally if a pull request is not merged.  
This is mainly about preserving the spirit of the project as an individual build.

If you are unsure, feel free to open an issue first before submitting a PR.

## Before Opening an Issue

If possible, please include:

- a clear description of the problem or suggestion
- steps to reproduce (for bugs)
- expected behavior
- actual behavior
- relevant logs or error messages
- environment details (OS, Python version, dependencies, etc.)

This makes it much easier to review and respond.

## Code Style (if submitting a PR)

If you do submit a pull request, please try to keep it:

- easy to review
- consistent with the existing project structure
- well explained in the PR description

Avoid unrelated changes in the same pull request.

## Respectful Collaboration

Please keep feedback constructive and respectful.

Even though this is a personal challenge project, thoughtful suggestions and discussions are always welcome and appreciated.

## Thank You

Thanks for checking out the project and taking the time to contribute ideas, feedback, or improvements.
